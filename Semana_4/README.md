# Semana 4 - Observabilidad, Shift-Right y Cierre QA

Esta semana contiene el material de clase de los dias 6 y 7.
La idea es cerrar el master conectando automatizacion, CI/CD, observabilidad y comunicacion tecnica.

## Estructura

- `dia-6-observabilidad-shift-right/`
  - Laboratorio forense para investigar un fallo productivo usando trazas y logs JSON.
  - Incluye guion del instructor, recursos de investigacion, taller teorico y solucionario.
- `dia-7-defensa-certificacion/`
  - Guia de defensa final, examen integrador y laboratorio practico extra.
  - El laboratorio extra convierte el cierre en una simulacion de war room QA.

## Orden recomendado para alumnos

1. Inicia por `dia-6-observabilidad-shift-right` para entender MTTR, trazas, metricas y logs.
2. Luego pasa a `dia-7-defensa-certificacion` para defender el proyecto y resolver el caso final.

## Nota de uso

Cada carpeta tiene su propio README con pasos concretos.
Los recursos estan pensados para correr en Windows con PowerShell, sin instalar dependencias.
